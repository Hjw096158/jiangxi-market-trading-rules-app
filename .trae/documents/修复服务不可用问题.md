## 修复服务不可用问题

### 问题分析
- 浏览器访问 http://localhost:5176/ 返回 "服务不可用"
- 开发服务器正在运行，但可能存在端口冲突或代码错误
- 之前修复了 HTML 实体问题，但可能还有其他问题

### 修复步骤

1. **停止所有运行的开发服务器**
   - 终止所有占用端口 5173-5176 的进程
   - 确保没有残留的开发服务器进程

2. **清除 Vite 缓存**
   - 运行 `npm run dev -- --force` 强制清除缓存并重启开发服务器
   - 这将确保使用最新的代码进行编译

3. **检查代码完整性**
   - 检查 App.tsx 中所有模拟器组件的导入是否正确
   - 确保所有模拟器组件都能正常编译
   - 特别检查 SupplyDemandCurveSimulator.tsx 是否还有其他错误

4. **验证开发服务器状态**
   - 检查开发服务器输出，确保没有编译错误
   - 确认服务器正在正确监听指定端口
   - 尝试访问服务器地址，确认服务可用

5. **测试模拟器功能**
   - 进入系统后，测试供需曲线模拟器是否能正常打开
   - 验证其他模拟器组件是否也能正常工作
   - 确保边际出清逻辑和坐标轴显示符合要求

### 预期效果
- 开发服务器能正常启动并监听端口
- 浏览器访问 http://localhost:5176/ 能正常显示页面
- 所有模拟器组件能正常加载和使用
- 供需曲线模拟器按照经济惯例显示（价格纵轴，数量横轴）

### 技术要点
- 确保所有 JSX 语法正确，不使用不必要的 HTML 实体
- 确保 Vite 开发服务器能正确编译所有代码
- 确保 React 组件能正常渲染和交互
- 确保供需曲线模拟器的逻辑和可视化符合要求
